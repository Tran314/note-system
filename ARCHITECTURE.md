## 架构特点

### 1. 本地优先 (Local-First)

所有数据操作首先在本地 IndexedDB 完成：
- **即时响应** - 无需等待网络请求
- **离线可用** - 无网络时完全可用
- **隐私安全** - 数据存储在用户浏览器

### 2. 按需同步 (Sync-on-Demand)

云端同步仅在用户主动触发时进行：
- **减少 API 调用** - 降低 COS 使用成本
- **冲突处理** - 本地版本优先
- **增量同步** - 仅同步变更数据

### 3. 无后端架构 (Backendless)

完全移除服务器端：
- **零服务器成本** - 无需维护后端
- **简化部署** - 纯静态文件托管
- **无限扩展** - 无服务器瓶颈

## 数据流

### 写入流程

```
用户操作 → Zustand Store → IndexedDB → 标记未同步
                                    ↓
                              用户触发同步
                                    ↓
                              COS 上传
                                    ↓
                              标记已同步
```

### 读取流程

```
用户请求 → 检查 IndexedDB 缓存
              ↓
         缓存命中？ ──Yes──→ 直接返回
              │
             No
              │
              ▼
         从 IndexedDB 读取
              │
              ▼
         返回数据
```

### 同步流程

```
触发同步 → 获取未同步数据 → 上传到 COS → 更新同步状态
              ↓
         检查云端更新 → 下载新数据 → 更新本地缓存
```

## 技术栈

### 前端框架
- **React 18** - UI 框架

### 状态管理
- **Zustand** - 轻量级状态管理

- **IndexedDB** - 浏览器本地数据库

### UI 组件
- **TailwindCSS** - 样式框架
- **TipTap** - 富文本编辑器
- **Lucide React** - 图标库

## 安全设计

### 本地安全
- **数据隔离** - 每个浏览器独立存储
- **无敏感数据** - 不存储密码等敏感信息
- **HTTPS** - 生产环境强制 HTTPS

### 云端安全
- **子账号密钥** - 限制 COS 访问权限
- **路径隔离** - 用户数据按路径隔离
- **环境变量** - 密钥不提交到代码库

---

## 性能优化

### 加载优化
- **代码分割** - 按需加载组件
- **骨架屏** - 提升感知速度
- **资源预加载** - 关键资源优先加载

### 存储优化
- **索引文件** - 快速列表查询
- **内存缓存** - 热点数据缓存
- **防抖更新** - 减少写入次数

## 部署架构

### 开发环境
```
本地开发服务器 (Vite) → IndexedDB (浏览器)
```

### 生产环境
```
CDN → 静态文件托管 → IndexedDB (浏览器)
                          ↓ 同步时
                      腾讯云 COS
```

### 支持平台
- **Web** - 所有现代浏览器
- **Desktop** - Electron 封装 (可选)
- **Mobile** - 响应式 PWA (计划中)

## 迁移历史

### v2.0 → v3.0 架构迁移

| 组件 | v2.0 | v3.0 |
|------|------|------|
| 后端 | NestJS | 移除 |
| 数据库 | PostgreSQL | IndexedDB |
| 缓存 | Redis | 内存缓存 |
| 认证 | JWT + bcrypt | 无 |
| 存储 | 本地文件 | 腾讯云 COS |
| 部署 | Docker + K8s | 静态托管 |
